##0x0. 前沿
我们在享受信息智能推送带来便捷的同时，更该警醒这种背后带来的片面。信息时代，接受的信息面很大程度将会塑造我们看待世界的方式。RSS是一种控制信息入口的坚守，但这种方式不断受到腐蚀，学会自己搭建，不仅能丰衣足食，更是一种“可控”的前提。
##0x1. 准备环境

  • 一台 Ubuntu 的服务器，这里安装的是ubuntu18.04LTE
  • 一个域名（非必须，最好提供）
  • 域名商处添加一条 DNS 的 A 记录指向服务器。

##1. 基础配置
开始前我们先更新下操作系统

###1.1. 安装docker
Docker 是非常优秀的虚拟化容器，借助于 Docker 我们可以方便的部署 Tiny Tiny RSS，首先我们在服务器上安装 Docker 本体。在服务器上面执行下面命令来安装 Docker：

因为docker进程中有管理员账户，运行docker时需要root权限，可以将普通用户添加到docker组内，这样就不用每次都切换到root用户了。这里以用户maverick为例

启动daocker

查看docker运行状态(ctl+C可以切回输入状态)

sudo curl -L https://github.com/docker/compose/releases/download/1.25.0/docker-compose-`uname -s-uname -m` -o /usr/local/bin/docker-compose

sudo chmod +x /usr/local/bin/docker-compose

mkdir ttrss && cd ttrss
curl -fLo docker-compose.yml https://github.com/HenryQW/Awesome-TTRSS/raw/master/docker-compose.yml

vim docker-compose.yml

docker-compose up -d

关闭 Docker 容器们

docker-compose down

删除已停止的 Docker 容器

docker-compose rm

再次开启 Docker 服务

docker-compose up -d

###2.2. Tiny Tiny Rss配置
打开浏览器ip:181，就可以看到Tiny Tiny Rss页面，默认账户：admin 密码：password，请第一时间在偏好设置——个人数据中修改
![15878943587371](/media/15878943587371.jpg)

导入订阅源，或者将其他RSS服务中的源导出，再导入Tiny Tiny RSS中，比如inoreader
![15878943773828](/media/15878943773828.jpg)

同样在偏好设置里，启动API（记得点击确认），并设置密码，这样就可以将账户添加到reeder3中，这样就可以愉快的玩耍了。
